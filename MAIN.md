# Invitation Management System

## 1. Overview

The system will be a web platform where users can **create, customize,
and manage digital invitations** for events.

Each invitation will function as a **customizable web page**, containing
text, images, buttons, and other event-related information.

The creator of the invitation will also be able to **control who can
access it** by linking guests to the invitation.\
When someone accesses the invitation link, they must **enter their email
address**, which will be validated against the guest list.

The system **will not send emails automatically**. Invitations will be
shared using **unique links generated by the platform**.

---

# 2. User Roles

## Invitation Creator (Platform User)

A person who registers in the system and manages invitations.

Permissions: - Create invitations - Edit invitations - Manage guest
lists - Customize invitation layout - Share invitation links - View RSVP
confirmations

## Guest

A person who receives the invitation link.

Possible actions: - Access the invitation link - Enter email to validate
access - View the invitation - Confirm attendance (optional)

---

# 3. Main System Flow

## 1. Account Registration

The user creates an account on the platform.

Basic information: - Name - Email - Password

---

## 2. Invitation Creation

The user creates a new invitation.

Example data:

- Event name
- Event date
- Event location
- Event description
- Cover image
- Invitation slug or URL

Example:

    https://invitation.app/ana-and-lucas-wedding

---

## 3. Invitation Customization (CMS)

Users can customize invitations using content blocks.

Examples of blocks:

### Text Block

    Welcome to our wedding!

### Image Block

Displays an uploaded image.

### Button Block

Examples:

    Confirm Attendance
    View Location
    Gift List

---

# 4. Guest Management

The invitation creator can manually add guests.

Example:

Name Email

---

John Smith john@email.com
Maria Souza maria@email.com

---

# 5. Access Control

When someone opens the invitation link:

### Step 1

The system asks for the guest's email.

    Enter your email to access the invitation

### Step 2

The system checks:

    Does the email exist in the guest list?

If **yes** → Access granted\
If **no** → Access denied

---

# 7. RSVP (Attendance Confirmation)

Guests may confirm attendance.

Example interface:

    Will you attend?

    [ YES ]   [ NO ]

Stored data example:

Guest Attendance

---

John Confirmed
Maria Declined

---

# 8. User Dashboard

The invitation creator can view event metrics.

Example:

- Total guests
- Confirmed attendees
- Declined attendees
- Pending responses

Example summary:

    100 guests
    ✔ 62 confirmed
    ❌ 8 declined
    ⏳ 30 pending
